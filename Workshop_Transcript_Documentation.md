### 3. Development Tools Setup

#### Code Editor vs IDE
- **VS Code**: Recommended lightweight code editor
  - Smaller file size
  - Sufficient for all development needs
  - Production-ready
- **PyCharm**: Full IDE option (not covered in installation)
  - More features but heavier
  - Advanced debugging capabilities

#### Programming Language
- **Python**: Chosen for Generative AI development
- **Why Python**: 
  - Large developer community
  - Extensive pre-built frameworks
  - Research and development primarily in Python
  - Reduces need to build everything from scratch

### 4. Python Installation Methods

#### Method 1: Direct Installation (python.org)
- Download Python as an application
- Similar to installing Spotify or other software
- Basic installation approach

#### Method 2: Package Manager (Anaconda)
- Two different Anaconda software options
- More comprehensive Python distribution

#### Method 3: Modern Approach (UV Package Manager)
- **UV**: Modern, fast Python package manager
- Recommended for this course
- Better dependency management
- Improved performance over traditional tools

### 5. Virtual Environments

#### VENV (Virtual Environment)
- **Purpose**: Creates isolated Python environments
- **Functionality**: 
  - Copy of original Python installation
  - Separate lib folders for different projects
  - Project-level dependency isolation
- **Limitations**:
  - Cannot manage different Python versions across projects
  - Limited to the system's Python version
  - Suitable when Python version is known and consistent

#### Best Practices
- Use VENV when Python version requirements are clear
- Consider advanced tools for multiple Python version management
- Essential for professional development workflows
